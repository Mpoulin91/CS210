Q: Summarize the project and what problem it was solving.
A: This purpose of this project was to show examples of how money can grow over time through interest accumulations and consistent monthly deposits into a savings account. The program will help teach and show the rewards of    fiscal responsibility.

  
Q: What did you do particularly well?
A: The thing that I did particularly well was the generation of the user interface. I was able to match it to the examples provided while also adding my own personal touches to make it stand out from the minimum layout.


Q: Where could you enhance your code? How would these improvements make your code more efficient, secure, and so on?
A: The place where my code could be improved would be to update the final investment report to show a more detailed outline of how the initial investment and monthly deposits grew. My code broke it down into a report at the   end of every year so it could be extrapolated to show the growth every 6 months or even after every month to provide a more detailed example.
 

Q: Which pieces of the code did you find most challenging to write, and how did you overcome this? What tools or resources are you adding to your support network?
A: For this project the most challenging aspect was the class creation with the variable pointers and class function to transition the proper formula and variable values from the MyClass.cpp to the InvestApp.cpp.
  

Q: What skills from this project will be particularly transferable to other projects or course work?
A:The skills from this project that will be particularly transferable would be class creation, class void function implementation, variable pointers, and integrating multiple cpp files into program creation.
 
 
Q:How did you make this program maintainable, readable, and adaptable?
A: The inclusion of extensive in-line comments and proper spacing with uniform naming structures heavily increased the readability, adaptability, and maintainability of the program to the point where if someone else was to     take over the program they would have no problem understanding the logic and flow of the code in order to make changes and updates as necessary. The adaptability of the code was handled by giving the user.
